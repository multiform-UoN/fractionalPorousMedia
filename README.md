# fractionalPorousMedia

Python implementation of a first-order product-integration finite-difference
scheme for **Basset-type time-fractional advection-diffusion-reaction equations**
that arise from rigorous micro-macro homogenisation of transport in
heterogeneous porous media.

## Connected paper

Berardi, Garrappa, Icardi, Nuca  
*"From microstructure to memory: Basset-type fractional transport models  
in porous media"*

The code exactly reproduces the numerical results and figures of the paper.

---

## Quick Start

```python
from python.main_BassetEqn import BassetConfig, solve
import numpy as np

cfg = BassetConfig(
    alpha=0.5,
    T=5.0,
    delta_time=0.01,
    nu0=1.0,
    vel0=-1.0,          # flow to the left (paper examples)
    beta0=0.5
)

t, x, u = solve(cfg)          # uses eigenfunction IC by default
```

Or with one-liner overrides (fully backward compatible):

```python
t, x, u = solve(alpha=0.25, T=10.0, delta_time=0.005)
```

## Core API

### `BassetConfig`

A dataclass that holds **all** parameters. See the extensive docstring inside
`main_BassetEqn.py` for the physical meaning of every field.

Most important groups:

| Group             | Key fields                          | Meaning |
|-------------------|-------------------------------------|---------|
| Domain            | `xL`, `xR`, `T`                     | Spatial interval and final time |
| Discretisation    | `N_space`, `delta_time`, `advection`| Grid size and time step |
| Fractional model  | `alpha`, `phi0`, `beta0`            | Order + two storage coefficients |
| Transport         | `nu0`, `vel0`, `reac0`              | Diffusion / velocity / reaction |
| BCs (Robin)       | `zetaL,xiL, zetaR,xiR`              | `zeta·u_x + xi·u = 0` |
| IC / Source       | `initial_condition`, `forcing`      | Callables or `None` (eigen-IC) |

**Special note on `phi0`**  
When `phi0 != 1` the interior discrete equations become  
`phi0 · (∂t u + beta0 · ∂t^α u) = L u + f`.  
Both time-derivative terms receive the factor (this is a direct consequence of
the paper's definition `B = diag(β) M`). Boundary rows are left unscaled.

### `solve(config=None, **kwargs)`

```python
time, mesh_x, u = solve(config)           # preferred
time, mesh_x, u = solve(alpha=0.5, T=2.0) # also works
```

Returns:
- `time`  – 1-D array of length `N_time`
- `mesh_x`– 1-D spatial grid
- `u`     – 2-D array `(N_space, N_time)`

The solver is **deterministic** and uses a pre-factorised LU decomposition +
incremental history accumulation for good performance.

## Verification & Tests

A small but rigorous test suite lives in `test_basset.py`:

```bash
cd python
python -m unittest test_basset -v
```

It contains:

1. Basic smoke test (eigenfunction IC execution).
2. **Method of Manufactured Solutions** (MMS) that isolates the temporal
   error of the product-integration rule with a space-independent exact
   solution `u = t²`.  
   The test confirms the theoretical **first-order** convergence rate
   `O(Δt)` (observed rates ≈ 1.01 and 1.008).

The forcing bug that existed before the refactor (missing the current time
level in the integrated source term) is covered by this test.

## Key Numerical Features (current implementation)

- Vectorial Basset formulation → arbitrary Robin BCs treated uniformly.
- Rectangular product-integration (order 1, implicit, excellent stability).
- Pre-factorised linear solve (A is constant).
- O(Nt) incremental history (instead of O(Nt²)).
- Optional fast contiguous BLAS slices.
- Eigenfunction initial condition via custom shifted inverse iteration.
- Clean separation of configuration from algorithm.
